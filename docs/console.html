xbasip: X-BASIC like package for micropython-x68k

xbasipパッケージ

consoleモジュール

  • screen(disp_size, page_mode, res=1, disp_on=True)

    • スクリーンモードを設定します。
    • disp_size -- 表示画面サイズ
      • 0 : 256×256画素
      • 1 : 512×512画素
      • 2 : 768×512画素
    • page_size -- 実画面（ページ）のサイズおよび色数、ページ数
      • 0 : 1024×1024画素、16色、1ページ
      • 1 : 512×512画素、16色、4ページ
      • 2 : 512×512画素、256色、2ページ
      • 3 : 512×512画素、65536色、1ページ
    • res -- スキャン
      • 0 : Low
      • 1 : High
    • disp_on -- グラフィック画面の表示ON/OFF
      • True : 表示ON
      • False : 表示OFF

  • width(columns)

    • テキスト画面の表示文字数を設定します。
    • columns -- 表示桁数
      • 96 : 96×32文字
      • 64 : 64×32文字
    • テキスト画面は消去され、console(0, 31, 1)が設定されます。
    • テキストパレットはデフォルト値に設定されます。

  • console(start, lines, fkey)

    • テキスト画面のスクロール範囲を設定します。
    • start -- スクロール開始行（0～31）
    • lines -- スクロール行数（1～32）
    • fkey -- ファンクションキー表示
      • True : 表示ON
      • False : 表示OFF
    • ファンクションキー表示ONの場合、全行数32行のうち1行をファンクションキーの表示に使うため、スクロール範囲は最大31行となります。

  • cls(all_clear=False)

    • テキスト画面を消去します。
    • all_clear
      • True : テキスト画面/グラフィック画面を消去、スプライト/BGの表示OFF
      • False : テキスト画面のみ消去

  • locate(x, y, cur_on=None)

    • カーソルを指定されたキャラクタ座標位置へ移動します。
    • x -- X座標
    • y -- Y座標
    • cur_on -- カーソル表示スイッチ
      • True : カーソル表示ON
      • False : カーソル表示OFF
      • None : カーソル表示の状態を変更しない

  • pos()

    • 現在のカーソルのX座標を返します。

  • csrlin()

    • 現在のカーソルのY座標を返します。

  • color(pal)

    • テキストの色、属性を設定します。
    • pal -- 色/属性コード
      • 0 : 黒
      • 1 : 水色
      • 2 : 黄色
      • 3 : 白
      • 4～7 : ボールド
      • 8～11 : 反転
      • 12～15 : ボールド反転

  • tpalet(pal=None, color=None)

    • テキストのパレットを設定します。
    • pal -- パレット番号
      • 0～3 : テキストカラー
      • 4～7 : 4～7に同じカラーコードを設定
      • 8～15 : 8～15に同じカラーコードを設定
    • color -- カラーコード
      • 0～65535 : 64K色コード
    • カラーコードを省略すると現在設定されているカラーコードを返します。
    • すべての引数を省略するとパレットをシステム既定値に戻します。
    • パレット番号4～15はマウスカーソルやソフトキーボード、電卓で使用されます。
    • パレット番号4～15を個別に設定する場合は、tpalet2()を使用してください。

  • tpalet2(pal=None, color=None)

    • テキストのパレットを設定します。
    • pal -- パレット番号
      • 0～15 : テキストカラー
    • color -- カラーコード
      • 0～65535 : 64K色コード（0b_GGGGG_RRRRR_BBBBB_L：RGB各5ビット、L輝度1ビット）
    • カラーコードを省略すると現在設定されているカラーコードを返します。
    • すべての引数を省略するとパレットをシステム既定値に戻します。
    • パレット番号4～15についても個別にカラーコードを設定します。この設定はcolor()では有効になりませんが、tgraphモジュールでテキストプレーン2、3へ描画する際に活用できます。

  • inkey(wait=True)

    • キーボードから入力された１文字のキャラクタコードを返します。
    • wait
      • True : キー入力されるまで待ちます
      • False : キー入力されるまで待ちません。入力が無い場合は0を返します

  • inkeyS(wait=True)

    • キーボードから入力された１文字をstr型で返します。
    • wait
      • True : キー入力されるまで待ちます
      • False : キー入力されるまで待ちません。入力が無い場合は''を返します

  • inkey0()

    • キーボードから入力された１文字のキャラクタコードを返します。入力が無い場合は0を返します。
    • 戻り値 -- キャラクタコード

  • keyinp()

    • キーデータを読み込みます。キー入力されるまで待ちます。
    • 戻り値
      • 上位8bit : スキャンコード（キーコードグループ×8＋キー状態ビット）
      • 下位8bit : キャラクタコード
    • SHIFT、CTRL、OPT.1、OPT.2については離した時にもスキャンコードに0x80を足した値を返します。

  • keysns()

    • キーが入力されているか調べます。キーデータバッファの内容は変化しません。
    • 戻り値
      • 0 : キー入力無し
      • 0x1_???? : キー入力あり。0x????はキーデータ

  • sftsns()

    • シフトキーの状態を調べます。
    • 戻り値
      • bit 15～8 : 未定義、全角、ひらがな、INS、CAPS、コード、ローマ字、かな
      • bit 7～0 : CAPS、コード、ローマ字、かな、OPT.2、OPT.1、CTRL、SHIFT
    • bit 7～0はキーが実際に押されていれば1、押されていなければ0を返します。
    • bit 15～8はロック状態ならば1、そうでなければ0を返します。

  • bitsns(group)

    • キーコードグループの状態を調べます。
    • group -- キーコードグループ（0x0～0xe）
    • 戻り値
      • bit 7～0 : キー状態ビット（押されていれば1、押されていなければ0）

  • keyflush()

    • キーデータバッファをクリアします。

  • key(fn, buf=None)

    • ファンクションキーに文字列を定義します。
    • fn -- ファンクションキー番号
      • 0 : すべてのキー
      • 1～10 : F1～F10
      • 11～20 : SHIFT＋F1～F10
      • 21～24 : R_UP、R_DN、INS、DEL
      • 25～28 : ↑、←、→、↑
      • 29～32 : CLR、HELP、HOME、UNDO
    • buf -- 定義文字列
      • fn = 0 : 712バイト
      • fn = 1～20 : 32バイト
      • fn = 21～32 : 6バイト
    • 戻り値
      • 定義文字列を省略すると現在の定義状態を返します
    • end()関数で終了するとファンクションキー定義は実行前の状態に復帰します。

  • funckey_on()

    • ファンクションキー表示をONします。

  • funckey_off()

    • ファンクションキー表示をOFFします。

  • cursor_on()

    • カーソル表示をONします。

  • cursor_off()

    • カーソル表示をOFFします。

  • deffont(code, buf, font_size=1)

    • 外字フォントを定義します。
    • code -- 外字文字コード
      • 0xf400～0xf5ff : 半角外字
      • 0xeb9f～0xebfc、0xec40～0xec7e : 全角外字
    • buf -- フォントデータ（bytearrayもしくはbytes）。1バイトあたり8画素
    • font_size -- フォントサイズ
      • 1 : 8×16、16×16
      • 2 : 12×24、24×24
    • end()関数で終了すると外字フォント定義は実行前の状態に復帰します。
    • 半角外字（0xf400～0xf5ff）の第二バイトは、シフトJIS規格の範囲（0x40～0x7e、0x80～0xfc）で使用することをお勧めします。

  • getfont(code, buf=None, font_size=1)

    • フォントデータを取得します。
    • code -- 文字コード
    • buf -- データの格納先（bytearray）。省略時は内部で確保します
    • font_size -- フォントサイズ
      • 0 : 6×12、12×12
      • 1 : 8×16、16×16
      • 2 : 12×24、24×24
    • 戻り値 -- tupleで (w, h, buf)を返します
      • w : フォント幅
      • h : フォント高さ
      • buf : データ

  • beep()

    • ビープ音を鳴らします。

  • end(msg=None)

    • プログラムを終了します。カーソル表示、ファンクションキー表示、ファンクションキー定義、外字フォント定義を実行前の状態に戻します。
    • msg -- 終了メッセージ
      • 終了時に表示します

  • priority(sp, tx, gr)

    • 画面表示の優先度を設定します。
    • sp -- スプライト/BGの優先度
      • 0, 1, 2 : 0が最前面、2が最背面
    • tx -- テキスト/マウスカーソルの優先度
      • 0, 1, 2 : 0が最前面、2が最背面
    • gr -- グラフィックの優先度
      • 0, 1, 2 : 0が最前面、2が最背面
    • デフォルトはpriority(0, 1, 2)の状態です（SP＞テキスト＞グラフィック）。

  • crtmod(mode, disp_on=True)

    • CRTモードを設定します。
    • mode -- CRTモード
    • disp_on -- グラフィック画面の表示ON/OFF
      • True : 表示ON
      • False : 表示OFF

4プレーンのビットマップで構成されていますが、通常のテキスト表示にはプレーン0、1のみを使用し、プレーン2、3はマウスカーソルやソフトキーボードに割り当てられています。そのため、color()では実質4色しか使用できません。

キー入力

カーソルキーはファンクションキーとしてユーザー定義可能です。そのため、inkey()で読み出される値はユーザーにより異なってしまいます。プログラムの最初にkey()により再定義することで解決します。key()で定義したファンクションキーはend()で終了する際に元の定義に戻ります。下記の例はカーソルキーをテンキー相当に定義します。

key(25, "8") # UP
key(26, "4") # LEFT
key(27, "6") # RIGHT
key(28, "2") # DOWN

キー入力がバッファにたまって扱いにくい場合がありますが、inkey()の前にkeyflush()でバッファをクリアすることができます。

print("何かキーを押してください")
keyflush()
k = inkey()

inkey()系の関数は、キーデータバッファを経由する事やリピート動作によってタイムラグがあるなど、アクションゲームにはあまり向いていません。複数キーの同時判定もできるbitsns()が有効です。

外字フォント定義

外字フォントを定義することにより、print()を使ってユーザー定義のキャラクタを表示することが可能です。

deffont(0x0ec40, b"\xff"*8+b"\x77"*8+b"\x33"*8+b"\x11"*8)
print(chr(0xec40))

deffont()で定義した外字フォントはend()で終了する際に元の定義に戻ります。
